SPACE ALLOCATION PROBLEMS:

❌ Top sections [1-4] consume ~610px out of 900px total window height
❌ Tab View gets only ~290px, severely limiting table visibility
❌ Session Browser table is cramped (shares ~150px with filter/buttons)
❌ Analysis Data table has limited rows visible (~200px content area)
❌ Users must scroll or resize window frequently
❌ Cognitive overload with so many visible sections simultaneously

═══════════════════════════════════════════════════════════════

KEY WIDGETS TO PRESERVE:

✅ ClientSelectorWidget - works well, should stay global
✅ SessionBrowserWidget - fully functional, needs more space
✅ File loading sections - feature-rich with single/folder modes
✅ QTableView with proxy model - powerful filtering/sorting
✅ Statistics and logs - valuable information display
